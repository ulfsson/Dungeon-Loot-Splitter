Logic Explanation

Array Logic
The primary array stores objects for the loot that's been added by the form. Those objects are constructed from a custom class that include three separate fields for the name, the value, and the rarity number. The class also provides a couple of features for getting the value and the name of the quality/rarity integer, which is useful when looping through each one of them. I'd planned to include some checks within the class in a future revision, to prevent inputting invalid values upon construction.

Looping Logic
The only real loop in use is when rendering the loot table inside of the renderLoot() method. Each time a valid piece of loot has been added, the method is called which loops through each object in the loot list array, loads up the appropriate variables, and constructs an HTML template which is then pushed to the loot table.

Conditional Logic
The only pieces of conditional logic involve some sanity checks to make sure the user isn't entering invalid data into the form fields, such as negative party size, strings in number fields, blank entries, etc.

The button below will call a method to instantly load up the loot list table with random pieces of dungeon loot. Number, name, value, and rarity are all randomized. Useful for quickly testing loot splitting features.
